# Session Summary ŌĆö 2026-05-04

## TUI (Terminal User Interface) work on `container_rl/cli.py`

### 1. Keyboard input architecture rewritten
**Problem:** Terminal raw mode was toggled on/off on every keystroke via `_getch`/`_getch_timeout`, causing input to be lost (submenu keys didn't register, 'q' didn't quit).

**Solution:** Terminal enters **cbreak mode** once at the start of `play()` via `_enter_raw_mode()`
(tty.setcbreak), and exits in a `finally` block via `_exit_raw_mode()`.  `tty.setcbreak` keeps
output processing intact (OPOST stays on) so Rich's escape sequences render properly.
A single `_read_key(timeout)` function replaces all previous key-reading functions:
- `timeout=None` ŌåÆ blocking (used for game-over screen, submenus)
- `timeout=N` ŌåÆ polling (main loop polls every 50ms so the display stays live)

### 2. Submenu rendering fix
**Problem:** `_input_number` called `live.update()` but never `live.refresh()`.
With `auto_refresh=False` on the Live context, submenu prompts were invisible and
the terminal appeared frozen.

**Solution:** Added `live.refresh()` after `live.update()` in `_input_number()`.

### 3. Rich markup parsing
**Problem:** `_player_card` and the prompt panel in `_render_frame` used `Text()` which
treats `[red]Red[/red]` etc. as literal text instead of styled markup.

**Solution:** Changed to `Text.from_markup()` in `_player_card` (for all markup-containing lines:
factory colors, secret color, bold labels, etc.) and in the prompt panel.

### 4. Produce action ŌĆö per-color price selection
**Problem:** PRODUCE hardcoded all containers to price slot 4 ($5). No way to choose prices.

**Solution:**
- TUI: When pressing `3` (Produce), the user is prompted **per factory color**:
  *"Produce Red ŌĆö pick a price: $1 $2 $3 $4"*, then *"Produce Blue ŌĆö ŌĆ”"*, etc.
- Each color+price calls `env.step()` individually.
- Only the **first** produce call consumes an action slot; subsequent calls for other
  colors don't (enforced in `_advance_turn` via `was_produced` flag).
- `q`/`ESC` cancels the entire produce sequence.

### 5. Submenu UX improvements
- Single-digit auto-submit: when max_val Ōēż 9, pressing `1` immediately selects
  option 1 without needing Enter.
- `q`/`Q` cancels any submenu (same as ESC).
- `_input_choice` gained an optional `header` parameter for contextual info.

### 6. Factory cost
**Problem:** Factory cost was `(n+1)*2` ŌåÆ $2, $4, $6, $8. Should be $6, $9, $12.

**Solution:**
- Changed formula to `(n+1)*3` in `_factory_cost()` (container.py).
- TUI action help now shows: `[1]BuyFactory ($6)` (dynamic, depends on current player's factory count).
- Buy factory submenu header shows: *"Buy a factory ŌĆö cost: $6"*.

## Game logic changes (`container_rl/env/container.py`)

### ActionEncoder
- `produce_actions` changed from `PRODUCE_PRICE_CHOICES` (4) to `nc * PRODUCE_PRICE_CHOICES` (20).
  Encodes `(color, price_slot)` pairs.
- `encode(ACTION_PRODUCE, {'color': c, 'price_slot': s})` ŌåÆ `offset + c*4 + s`
- `decode()` extracts both color and price_slot.

### `_action_produce`
- Now produces **one color at a time** (from `HEAD_COLOR`) instead of iterating all colors.
- `can_produce` no longer checks `already_produced` (allowing multiple calls per turn).
- Payment ($1) only happens on the first produce (when `produced_this_turn == 0`).

### `_advance_turn`
- Accepts `was_produced` flag (captured before dispatch) so it knows whether
  this is the first produce of a batch.  Only the first counts as an action.

### `_compute_offsets_array`, `_flat_to_multihd`, `to_multi_head`
- All updated for the new `(color, price_slot)` produce encoding.

### `_factory_cost`
- Changed from `(num_factories_owned + 1) * 2` to `(num_factories_owned + 1) * 3`.
- New costs: 2nd factory $6, 3rd $9, 4th $12.

### Constants added
- `PRODUCE_PRICE_CHOICES = 4` (price slots 0ŌĆō3 for $1ŌĆō$4).
